## 新增需求

### 需求: 生成分析结果包
系统必须在视频处理完成后生成结构化分析结果包，除热力图外，还应包含车型结构、分时段流量、轨迹摘要、热点区域和处理性能等结果。分析结果必须保存到结果目录，便于论文和答辩材料引用。

#### 场景: 完成视频处理
- **当** 用户完成一次视频处理
- **那么** 系统在结果目录中保存分析摘要文件和可选图表/CSV/JSON 数据

### 需求: 输出车型结构分析
系统必须输出各车辆类别的累计数量和占比，类别名称必须来自统一 taxonomy。

#### 场景: 车型统计
- **当** 视频处理完成
- **那么** 分析结果包含每类车辆数量、总车辆数和每类占比

### 需求: 输出分时段流量趋势
系统必须按视频时间分桶统计车辆流量，输出总流量趋势和分车型流量趋势。

#### 场景: 分钟级流量趋势
- **当** 用户设置时间桶大小为 60 秒
- **那么** 系统输出每分钟的总车辆数和各车型车辆数

### 需求: 输出轨迹摘要
系统必须为每个有效 `track_id` 输出轨迹摘要，包括类别、出现帧范围、持续时长、轨迹点数和像素路径长度。若没有物理标定，系统不得把像素路径长度声明为真实行驶距离或真实车速。

#### 场景: 轨迹摘要导出
- **当** 视频处理完成
- **那么** 系统输出每辆车的轨迹摘要记录

### 需求: 输出热点区域排名
系统必须基于热力图累计数据统计高密度区域。区域统计可以采用网格划分方式，输出 Top N 热点区域的位置和累计强度。

#### 场景: 交通热点分析
- **当** 热力图累计完成
- **那么** 系统输出车辆活动最密集的区域排名

### 需求: 输出处理性能摘要
系统必须记录处理性能信息，包括视频总帧数、处理耗时、平均 FPS、模型路径和计算设备。

#### 场景: 实时性分析
- **当** 视频处理完成
- **那么** 系统输出可用于论文实时性分析的性能摘要
