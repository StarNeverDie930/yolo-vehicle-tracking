## 新增需求

### 需求: README 与论文 markdown 同步实现
README 必须与真实实现保持一致，尤其是框架名称、类别命名、计数定义、热力图定义和评估范围。任务书 markdown、开题报告 markdown 作为基线材料只进行复核，不直接修改正文。

#### 场景: 框架名称校验
- **当** README 描述 GUI 所使用的框架
- **那么** README 必须写成与代码一致的 PySide6，而不是 PyQt6

### 需求: 文档仓库规则允许跟踪 markdown deliverables
Markdown 版毕业设计文档必须可以进入版本控制；`docs/` 目录不得再被整目录忽略。若需要保留旧版二进制源文件，只能使用更细粒度的忽略规则。

#### 场景: 新文档提交
- **当** 新的 `docs/任务书.md` 或 `docs/开题报告.md` 已作为基线材料存在
- **那么** 文件能够被正常纳入仓库并参与版本管理

### 需求: 文档不得声明未实现功能
README 中对计数、热力图、评估和导出能力的描述必须与当前实现或已批准的变更保持一致，不得把未实现的功能写成已完成。任务书和开题报告只记录复核结论，不修改正文。

#### 场景: 论文材料复核
- **当** 开题报告或任务书被再次阅读
- **那么** 文档中的术语、功能边界和验证范围应与代码实现一致
