## 新增需求

### 需求: 每次评估生成独立结果包
系统必须为每次评估生成独立的时间戳结果目录，目录中包含摘要文件和图表文件，不得被后续运行覆盖。

#### 场景: 同一序列重复评估
- **当** 用户对同一序列运行两次评估
- **那么** `evaluation/results/` 下存在两个不同的时间戳目录，各自独立保留

### 需求: 摘要文件包含完整元信息
摘要文件必须包含模型路径、序列名、运行模式、时间戳、环境信息，以及所有已执行评估的指标数值。

#### 场景: 导出论文材料
- **当** 评估结束后生成摘要文件
- **那么** 用户可以直接引用该摘要文件中的模型、指标和运行信息进入论文或答辩材料

### 需求: 评估覆盖公开序列与自采视频
系统必须提供公开数据集序列和自采视频的评估记录或入口，不能只保留单一序列的结果。

#### 场景: 自采视频验证
- **当** 用户提供一个自采视频样本
- **那么** 系统能够输出与公开序列同格式的统计结果、摘要和图表

### 需求: 图表生成失败不阻断评估
若图表生成失败，系统必须记录警告，但摘要文件和主评估流程必须继续完成。

#### 场景: matplotlib 不可用
- **当** 图表生成阶段抛出异常
- **那么** 系统输出警告并继续写入摘要文件
