## 新增需求

### 需求: 处理失败可见并可恢复
系统必须在视频打开、读帧、写入、导出或模型切换失败时向用户显示明确错误信息，并恢复 GUI 控件状态。

#### 场景: 视频打开失败
- **当** 用户选择了一个无法打开的视频文件
- **那么** 系统提示失败原因，不进入处理状态

#### 场景: 处理中发生异常
- **当** 推理或写入过程中抛出异常
- **那么** 系统停止当前任务，恢复按钮状态，并提示用户

### 需求: 输出写入器必须先通过校验
视频导出前必须校验输出写入器是否成功打开；如果编码器或路径不可用，系统必须立即失败并给出可理解的提示。

#### 场景: 编码器不可用
- **当** 输出编码器与目标容器不兼容
- **那么** 系统不应静默继续，而应提示用户更换输出格式或路径

### 需求: 模型切换与处理互斥
系统必须保持“处理过程中不能切换模型”的互斥约束，避免运行状态混乱。

#### 场景: 正在处理视频时切换模型
- **当** 用户在处理进行中点击模型切换
- **那么** 系统拒绝切换并提示先停止当前任务
