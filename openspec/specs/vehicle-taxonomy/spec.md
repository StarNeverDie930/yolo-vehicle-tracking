## 新增需求

### 需求: 车辆类别存在单一来源
系统必须从单一的 taxonomy 定义读取车辆类别顺序和显示名称，检测、统计、绘制、训练配置、评估摘要和文档不得再各自维护不同的类别表。

#### 场景: 自训练模型与界面显示一致
- **当** 系统加载自训练四类模型
- **那么** 画框标签、统计面板、评估摘要和数据集 yaml 使用同一组类别顺序

### 需求: 源数据映射到统一四类
系统必须把 UA-DETRAC、KITTI 等来源标签映射到统一的四类 taxonomy。第四类作为合并后的大车类/第四类，在代码、配置和文档中保持同一命名，不得在运行时混用 `others`、`truck` 或 COCO 类别编号。

#### 场景: 合并训练集
- **当** 生成训练集和 merged.yaml
- **那么** 输出标签与运行时类别顺序一致

### 需求: 禁止绘制层硬编码 COCO 类别
绘制层不得再依赖与当前模型无关的 COCO 类别编号表；对自训练四类模型，绘制层必须使用统一 taxonomy 中的名称。

#### 场景: 结果视频标注
- **当** 系统导出结果视频
- **那么** 视频中的类别标签与统计面板中的类别标签保持一致
