## 新增需求

### 需求: 后台线程必须安全结束
系统必须区分业务处理完成和 QThread 真实结束，且不得在线程仍运行时销毁 worker 对象。

#### 场景: 正常处理完成
- **当** 用户启动视频处理并处理到最后一帧
- **则** worker 必须先完成视频资源释放和线程退出流程
- **并且** 主窗口不得在线程仍运行时将 worker 引用置空
- **并且** 终端不得出现 `QThread: Destroyed while thread is still running`

#### 场景: 业务信号命名
- **当** worker 通知主窗口处理完成、取消或失败
- **则** 自定义业务信号不得与 `QThread.finished` 同名
- **并且** `QThread.finished` 只用于线程真实结束后的清理动作

### 需求: 处理完成后控件状态必须正确恢复
系统必须在处理完成后显示完成提示，并进入可继续操作的完成状态。

#### 场景: 处理完成提示
- **当** 视频处理正常完成
- **则** 系统必须弹窗提示处理完成
- **并且** 提示中应包含分析结果保存目录
- **并且** 进度应显示为 100%

#### 场景: 完成后的按钮状态
- **当** 视频处理正常完成
- **则** 停止按钮必须置灰
- **并且** 用户必须可以导出视频
- **并且** 用户必须可以打开另一个视频
- **并且** 用户必须可以查看热力图
- **并且** 用户必须可以保存热力图

### 需求: 处理完成但视频未导出时必须提示
系统必须记录普通处理完成后的视频导出状态，并在用户打开新视频前提示未导出风险。

#### 场景: 普通处理完成后未导出
- **当** 用户完成一次普通视频处理
- **并且** 用户尚未导出处理后视频
- **当** 用户点击打开另一个视频
- **则** 系统必须弹窗提示 `您还有一个处理完成的视频未保存，是否要继续？`

#### 场景: 用户取消继续
- **当** 未导出提示弹出
- **并且** 用户选择不继续
- **则** 系统不得打开视频选择窗口
- **并且** 当前处理结果状态必须保留

#### 场景: 用户确认继续
- **当** 未导出提示弹出
- **并且** 用户选择继续
- **则** 系统必须打开视频选择窗口
- **并且** 用户选中新视频后必须清理旧处理状态

#### 场景: 已导出后打开新视频
- **当** 当前处理结果已经导出为视频文件
- **当** 用户点击打开另一个视频
- **则** 系统不应显示未保存视频提示
- **并且** 应直接进入视频选择流程

### 需求: 导出视频期间必须进入 busy 状态
系统必须在导出视频期间禁止会破坏当前后台任务的一切操作。

#### 场景: 导出进行中
- **当** 用户开始导出处理后视频
- **则** 系统必须禁用打开视频、开始处理、再次导出、查看热力图和保存热力图操作
- **并且** 系统必须显示当前处于导出或忙碌状态

#### 场景: 导出完成
- **当** 视频导出正常完成
- **则** 系统必须弹窗提示导出完成
- **并且** 记录导出视频路径
- **并且** 将当前处理结果标记为已保存视频
- **并且** 恢复完成后的可操作状态

### 需求: 用户停止必须区别于正常完成
系统必须区分用户主动停止和正常处理完成，避免取消后误报完成。

#### 场景: 用户停止处理
- **当** 用户在处理或导出期间点击停止
- **则** 系统必须进入停止中状态
- **并且** 停止按钮必须置灰以避免重复停止请求
- **并且** worker 必须尽快安全退出

#### 场景: 停止后退出
- **当** worker 因用户停止而退出
- **则** 系统不得显示正常处理完成提示
- **并且** 应显示取消或已停止语义
- **并且** 应恢复到可继续操作的状态

### 需求: 关闭窗口时必须保护后台任务
系统必须在窗口关闭时检查后台线程状态，避免关闭窗口导致 QThread 被强制销毁。

#### 场景: 空闲时关闭
- **当** 没有后台 worker 正在运行
- **并且** 用户关闭主窗口
- **则** 应正常关闭程序

#### 场景: 后台任务运行时关闭
- **当** 处理或导出 worker 仍在运行
- **并且** 用户关闭主窗口
- **则** 系统必须弹窗提示后台任务仍在运行
- **并且** 用户取消关闭时程序必须保持运行
- **并且** 用户确认关闭时系统必须请求 worker 停止并等待线程安全退出

### 需求: 热力图查看和保存必须分离
系统必须将查看热力图和保存热力图作为两个明确动作，避免查看动作产生不必要文件。

#### 场景: 查看热力图
- **当** 视频处理已经完成并且存在热力图数据
- **并且** 用户点击查看热力图
- **则** 系统必须在界面中显示叠加热力图
- **并且** 不应写入新的热力图文件

#### 场景: 保存热力图
- **当** 视频处理已经完成并且存在热力图数据
- **并且** 用户点击保存热力图
- **则** 系统必须保存原始热力图和 overlay 热力图到当前结果目录
- **并且** 弹窗提示两个文件路径
