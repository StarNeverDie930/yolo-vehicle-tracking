## 新增需求

### 需求: 累计计数和越线流量分开
系统必须同时支持 `unique_track` 和 `line_crossing` 两种统计模式。`unique_track` 按 `track_id` 去重统计累计车辆数；`line_crossing` 统计穿越虚拟检测线的车辆数。

#### 场景: 默认累计计数
- **当** 用户未显式选择线检测模式
- **那么** 系统按 `unique_track` 模式输出累计车辆数

#### 场景: 越线流量统计
- **当** 用户启用 `line_crossing` 模式
- **那么** 系统根据 `line_position` 和 `direction` 统计通过检测线的车辆数

### 需求: 越线计数必须只统计真实跨线轨迹
系统必须在 `line_crossing` 模式下根据车辆中心点跨越虚拟检测线来计数，而不是仅因为车辆中心点触碰检测线就计数。

#### 场景: 水平检测线跨越
- **当** 车辆轨迹中心点从水平检测线的一侧移动到另一侧
- **则** 系统必须将该 `track_id` 计入越线车辆数
- **并且** 对应车型统计和流量窗口统计必须同步增加

#### 场景: 垂直检测线跨越
- **当** 车辆轨迹中心点从垂直检测线的一侧移动到另一侧
- **则** 系统必须将该 `track_id` 计入越线车辆数
- **并且** 对应车型统计和流量窗口统计必须同步增加

#### 场景: 未跨越检测线
- **当** 车辆轨迹中心点始终位于检测线同一侧
- **则** 系统不得增加越线车辆数

#### 场景: 同一轨迹重复跨线
- **当** 同一个 `track_id` 已经被记录为越线车辆
- **并且** 后续该轨迹再次跨越检测线
- **则** 系统不得重复增加越线车辆数

### 需求: 计数模式切换必须保持统计语义一致
系统必须在 `unique_track` 和 `line_crossing` 两种计数模式之间保持清晰边界，避免不同模式下的历史统计混合显示。

#### 场景: 默认 ID 去重计数
- **当** 用户未显式启用越线计数
- **则** 系统必须使用 `unique_track` 作为默认计数模式
- **并且** 首次出现的合法车辆 `track_id` 必须计入累计车辆数

#### 场景: 切换计数模式
- **当** 用户在空闲状态下切换计数模式
- **则** 系统必须同步运行配置和当前 `VehicleCounter`
- **并且** 后续处理结果必须使用新的计数模式
- **并且** 不得将切换前的统计数据混入切换后的统计结果

#### 场景: 越线参数变更
- **当** 用户在空闲状态下修改越线方向或越线位置
- **则** 后续越线计数必须使用新的检测线参数
- **并且** 检测线绘制位置必须与计数参数一致

### 需求: 流量窗口基于视频时间
近 N 秒流量必须基于视频时间计算，而不是基于处理线程的墙钟时间。若源视频 FPS 缺失或异常，系统必须使用可配置的兜底值并在摘要中记录。

#### 场景: 视频播放速度变化
- **当** 视频处理速度快于或慢于真实播放速度
- **那么** 近 1 分钟流量仍按视频时间计算，结果不会因为机器性能而漂移

### 需求: 热力图使用完整运行轨迹累积
系统必须为热力图维护独立的累计数据结构。轨迹显示缓存可以保留固定长度，但热力图必须反映当前运行过程中的全部有效轨迹点。

#### 场景: 处理完成后查看热力图
- **当** 视频处理结束并生成热力图
- **那么** 热力图展示的是整段运行过程中车辆中心点的空间分布，而不是仅显示最近若干帧

### 需求: 热力图可保存
系统必须支持将热力图保存到结果目录。保存内容至少包括原始热力图图片和叠加到代表帧上的 overlay 热力图图片，文件名必须避免覆盖历史结果。

#### 场景: 保存热力图
- **当** 用户在处理完成后点击保存热力图或生成结果报告
- **那么** 系统在结果目录中写入原始热力图和 overlay 热力图图片

#### 场景: 重复保存
- **当** 用户对同一视频多次保存热力图
- **那么** 系统使用时间戳或序号区分输出文件，不覆盖已有图片
